# 第一期知识库验收追溯

本文记录当前实现与一期 V1.1 需求的对应关系。知识库前端保持单一入口和文件树布局；Phase 1 资料仅通过网页/API 上传接入并写入系统默认知识库目录，未来 Wiki 等来源通过标准 Source Connector 扩展，不包含本地目录监听。

| 范围 | 实现证据 | 自动化证据 |
|---|---|---|
| FR-001～002 默认知识库与概览 | `KnowledgeService.createProject`、`overview`、`GET /overview` | AC-001 |
| FR-003～005 配置版本与影响分类 | `saveConfig`、兼容指纹、无变化短路、完整知识库配置页 | FR-003/004、AC-008 |
| FR-006～009 资料接入 | `ingest` 只固化快照并创建 `queued` 任务、上传接口、未来连接器边界 | AC-002/003/005、异步上传测试 |
| FR-010～011 资产生命周期 | 稳定资产、不可变版本、Hash 去重与删除对账 | AC-004、删除测试 |
| FR-012～016 同步与索引 | 持久化 Worker、稳定 Chunk、Embedding 复用、独立 IndexChunk、候选校验和条件切换 | AC-004/005/006/008、处理中取消测试 |
| FR-017～019 浏览与检索 | 唯一知识库文件树、真实原文/历史、顶部混合检索与固定版本证据 | AC-007、AC-009 |
| FR-020～022 重建与可观测性 | 异步重建、进度轮询、取消、失败保旧索引、任务恢复与重试 | FR-020/021、AC-005/008 |
| PostgreSQL 持久化 | 事务锁内读取最新状态、实体级 UPSERT/定向删除、索引固定 Chunk 表 | 构建与迁移检查；真实数据库集成测试待部署环境执行 |
| 检索降级 | 混合检索向量失败时退回关键词；纯向量返回明确不可用状态；Reranker 可降级 | 检索降级自动化测试 |

## 验证命令

```powershell
$ErrorActionPreference = 'Stop'
npm test
npm run build
git diff --check
```

浏览器联调执行一次 `npm run dev`，会同时启动 API 与 Web。配置 `DATABASE_URL` 时，知识库状态保存在 PostgreSQL 的 `smarthub` schema；未配置时回退到 `data/smarthub.json`。上传原文件和模型缓存仍写系统默认目录，运行数据不提交 Git。
