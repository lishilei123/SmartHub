# 第一期知识库验收追溯

本文记录当前实现与知识库产品原型的对应关系。用户最新确认知识库前端保持单一入口和原文件树布局，不展示额外筛选栏；资料仅通过上传接入，并写入系统默认知识库目录。

| 范围 | 实现证据 | 自动化证据 |
|---|---|---|
| FR-001～002 默认知识库与概览 | `KnowledgeService.createProject`、`overview`、`GET /overview` | AC-001 |
| FR-003～005 配置版本与影响分类 | `saveConfig`、兼容指纹、无变化短路、完整知识库配置页 | FR-003/004、AC-008 |
| 资料接入 | `ingest`、上传接口、系统默认知识库目录、不可变原文件快照 | AC-002/003、文件落盘测试 |
| FR-010～011 资产生命周期 | 稳定资产、不可变版本、Hash 去重与删除对账 | AC-004、删除测试 |
| FR-012～016 同步与索引 | 稳定 Chunk、Embedding 复用、任务记录、候选索引和活动指针切换 | AC-004/006/008 |
| FR-017～019 浏览与检索 | 唯一知识库文件树、真实原文/历史、顶部混合检索与固定版本证据 | AC-007、AC-009 |
| FR-020～022 重建与可观测性 | 异步重建、进度轮询、取消、失败保旧索引、任务列表与重试 | FR-020/021、AC-008 |

## 验证命令

```powershell
$ErrorActionPreference = 'Stop'
npm test
npm run build
git diff --check
```

浏览器联调执行一次 `npm run dev`，会同时启动 API 与 Web。配置 `DATABASE_URL` 时，知识库状态保存在 PostgreSQL 的 `smarthub` schema；未配置时回退到 `data/smarthub.json`。上传原文件和模型缓存仍写系统默认目录，运行数据不提交 Git。
